egrep | grep 差别以及 egrep 用法

发表于 2016-04-15   |   分类于 Basics   |  

简介

  • 基于 “PATTERN” 对给定文本进行搜索操作,相当于 “ grep -E “ ;

  • grep详情参考：( grep 相应的正则表达式和用法 )

语法及参数

egrep [OPTIONS] PATTERN [FILE...]
      egrep [OPTIONS] [-e PATTERN | -f FILE] [FILE...]

常用选项：
    --color=auto：对匹配到的文本高亮显示;
              -i：忽略字符大小写;
              -o：仅显示匹配 到的文本自身;
              -v：反向匹配;
              -q：静默模式，不输出任何信息;
           -r/-d：递归;
              -H：--with-filename 显示匹配行,并且列出所属的文件名称。
         -f FILE：--file=FILE:FILE为每行包含了一个pattern的文件,即 grep script;
          -A NUM：显示匹配行,并且显示向后的 "NUM" 行;
          -B NUM：显示匹配行,并且显示向前的 "NUM" 行;
          -C NUM：显示匹配行,并且显示前后各 "NUM" 行;
          --help：获取帮助

正则表达式用法：
    匹配次数：
        用在要指定其出现的次数的字符后面，用限制其前面的字符要出现的次数；默认工作于贪婪模式;
              []：匹配范围内的任意单个字符;
             [^]：匹配范围外的任意单个字符;
               *：匹配前面的字符任意次;
              .*：匹配任意长度的任意字符;
               +：匹配前面的字符至少1次;
               ?：匹配前面的字符0次或1次;
           \{m\}：其前面的字符出现m次,m为非负整数;
         \{m,n\}：其前面的字符出现m~n次;         ## \{m,\}:至少m次;  \{0,n\}:最多n次;
     [[:digit:]]：所有数字;          #[0-9]
     [[:alpha:]]：所有字母;          #[a-zA-Z]     
     [[:lower:]]：所有小写字母;      #[a-z]
     [[:upper:]]：所有大写字母;      #[A-Z]
     [[:alnum:]]：所有字母和数字;    #[a-zA-Z0-9]
     [[:space:]]：空白字符;
     [[:punct:]]：标点符号;

位置锚定：
            ^：行首锚定;
            $：行尾锚定;
        \<,\b：词首;
        \>,\b：词尾;

分组与引用：
    (PATTERN)：分组括号中匹配到的字符会被正则表达式引擎自动记录于内部的变量中;
     后向引用：\1,\2,\3,...
或者：
        x|y：表示"x"或者"y"
      Z|zsh：表示"Z"或者"zsh"
    (Z|z)sh：表示"Zsh"或者"zsh"

总结 "grep" | "egrep" 区别：

    grep egrep

    相同
    \<
    \>

    不相同
    grep
        \+
        \*
        \{\}
        \|
    -E    (s|S)
    egrep
        +
        *
        {}
        |
        (s|S)

实例

1、找出”ldd /usr/bin/cat”命令的结果中文件路径；

2、找出/proc/meminfo文件中，所有以大写或小写s开头的行；至少用三种方式实现；

3、查看当前系统上root、bash或nogloin用户shell是否为用户名；

4、echo输出一个绝对路径，使用egrep取出其基名；

5、找出ifconfig命令结果中的1-255之间的整数；

6、添加用户bash、testbash、basher及nologin，前三个用户的默认shell为/bin/bash，nologin的默认shell为/sbin/nologin，而后找出其用户名与shell名相同的用户；
